# WordQuest — 작업 규칙 (모든 세션 공통)

> 이 저장소에서 작업하는 **모든 세션**이 시작 시 자동으로 읽는 규칙 파일이다.
> 프로젝트 맥락·변경 히스토리·시스템 설명은 `HANDOFF.md` 를 참고한다.

---

## 필수 작업 절차 (의무)

모든 실질적 작업은 아래 **4단계 사이클**을 반드시 순서대로 거친다. 건너뛰지 않는다.

1. **문제 식별 → 해결책 설계 (Plan)**
   문제를 명확히 정의하고, 해결 방안을 계획으로 정리한다.
2. **독립 에이전트의 Plan 검수**
   구현에 착수하기 **전에**, 작성자가 아닌 **독립된 서브에이전트**가 계획을 검수한다.
3. **구현 (Implementation)**
   검수를 통과한 계획대로 구현한다.
4. **독립 에이전트의 구현 검수**
   구현 결과를 다시 **독립된 서브에이전트**가 검수한다.

```
문제 식별·Plan  →  [독립 검수]  →  구현  →  [독립 검수]  →  완료
                     ≥85 통과              ≥85 통과
```

### 검수 통과 기준
- 검수는 **100점 만점**으로 채점한다.
- **85점 이상**이어야 통과. **85점 미만이면** 지적사항을 반영해 해당 단계(Plan 또는 구현)를 고치고 **재검수**한다(통과할 때까지 반복).
- **2단계(Plan 검수)와 4단계(구현 검수) 모두** 85점 이상을 받아야 작업을 완료로 간주한다.

### 원칙
- 검수는 반드시 **독립된 에이전트**가 수행한다(작성자 자신이 채점하지 않음) → 객관성 확보.
- 검수 결과(점수·지적사항·통과 여부)는 사용자에게 요약해 보고한다.
- 코드·로직·데이터·보안·배포에 영향을 주는 변경은 **예외 없이** 이 사이클을 따른다.
  (오타·주석·순수 문서 수정처럼 위험이 없는 자명한 변경은 생략할 수 있다.)

## 동시 작업 주의
- 이 저장소는 여러 세션(에이전트)이 **동시에** 작업할 수 있다.
- 작업 전 `git fetch` 로 최신 상태를 확인하고, 다른 브랜치가 만지는 파일과 **겹치지 않게** 한다.
- 자기 지정 브랜치에서 작업하고, `main` 병합 전에는 항상 최신 `main` 을 확인한다.
